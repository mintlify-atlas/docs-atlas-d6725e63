This guide walks you through a complete logo generation session using a concrete example: creating a logo for **DataFlow**, an AI data pipeline tool. By the end, you will have SVG variants, an interactive HTML showcase, and (optionally) high-end showcase images generated by Gemini.

Before starting, make sure you have [installed the skill and configured your environment](/installation).

## Generate your first logo

<Steps>
  <Step title="Start a new conversation in Claude Code">
    Open Claude Code and activate the skill by typing `/logo-generator`, or simply describe what you want. The skill activates whenever you ask for a logo. For example:

    ```
    Generate a logo for my AI product called "DataFlow"
    ```

    Claude confirms the skill is active and moves into Phase 1.
  </Step>

  <Step title="Answer the information-gathering questions (Phase 1)">
    Claude asks a series of short questions to understand your product before generating anything. Questions cover four areas:

    - **Product name** — already provided: "DataFlow"
    - **Industry / category** — for example: `AI infrastructure / data engineering`
    - **Core concept** — the idea the logo should communicate, for example: `flow, connection, transformation`
    - **Design preferences** — style (minimal vs. complex), color palette (monochrome vs. specific), mood (cold/technical vs. warm/approachable)

    A typical exchange looks like this:

    ```
    Claude: What industry or category does DataFlow belong to?
    You:    AI data pipelines — developer tooling for ETL and data transformation

    Claude: What core concept should the logo communicate?
    You:    Flow and connection — data moving through a system

    Claude: Any color or style preferences?
    You:    Monochrome, minimal, technical feel
    ```

    Claude asks concisely and does not ask for more information than it needs.
  </Step>

  <Step title="Review the SVG variants (Phase 2)">
    Claude generates six or more distinct SVG variants. Each variant uses `viewBox="0 0 100 100"` so the designs scale perfectly at any size. Variants deliberately differ in pattern type — not just parameter tweaks — so you are evaluating genuinely different directions.

    For a product like DataFlow, you might see variants based on patterns like these:

    **Node network** — nodes connected by paths, suggesting data flowing through a pipeline:

    ```svg
    <svg viewBox="0 0 100 100">
      <path d="M 30 70 Q 50 70, 50 50 T 70 30" stroke="currentColor" stroke-width="2" fill="none"/>
      <circle cx="30" cy="70" r="4" fill="currentColor"/>
      <circle cx="50" cy="50" r="5" fill="currentColor"/>
      <circle cx="70" cy="30" r="4" fill="currentColor"/>
    </svg>
    ```

    **Geometric shape with line accent** — a triangle with a highlighted node, suggesting direction and precision:

    ```svg
    <svg viewBox="0 0 100 100">
      <polygon points="50,30 70,60 30,60" fill="none" stroke="currentColor" stroke-width="2"/>
      <circle cx="50" cy="30" r="4" fill="currentColor"/>
    </svg>
    ```

    **Concentric circle dots** — rings of dots suggesting orbital systems or data layers:

    ```svg
    <svg viewBox="0 0 100 100">
      <g>
        <!-- Inner ring: 6 dots -->
        <circle cx="50" cy="38" r="3" fill="currentColor"/>
        <!-- Additional dots arranged in circular rings -->
      </g>
    </svg>
    ```

    Along with each variant, Claude explains the design rationale — why this pattern fits DataFlow's concept of data flow and transformation.

    Immediately after the variants, Claude generates an **interactive HTML showcase** file. Open it in a browser to see all variants on a dark background with hover effects and smooth transitions, making it easy to compare them side by side.
  </Step>

  <Step title="Select favorites and iterate (Phase 3)">
    Tell Claude which variant or variants you prefer, and describe any changes you want. You can:

    - Select a variant by number: `I like variant 3`
    - Request a parameter change: `Make the stroke weight slightly heavier`
    - Ask for a combination: `Can you combine the node pattern from variant 2 with the spacing from variant 5?`
    - Request a new direction: `Generate two more variants exploring just the flow concept, no nodes`

    Claude makes targeted adjustments and does not regenerate the full set unless you ask for it.

    ```
    I like variant 3. Can you increase the spacing between the nodes and
    add a faint second path line behind the main one?
    ```

    Iterate until you have a design you want to present professionally.
  </Step>

  <Step title="Generate high-end showcase images (Phase 4)">
    Once you are happy with a variant, ask Claude to proceed to showcase generation:

    ```
    I'm happy with variant 3. Generate showcase images for it.
    ```

    Claude will:

    1. Export the SVG to a 1024×1024px PNG with a transparent background using `scripts/svg_to_png.py`
    2. Recommend four background styles suited to DataFlow's technical, AI-native character — for example, **Fluid Abyss** (deep purple, AI-native), **The Void** (absolute black, hardcore tech), **Clinical Studio** (pure white, algorithm-driven), and **UI Container** (frosted glass, SaaS native)
    3. Run `scripts/generate_showcase.py` with your Gemini API key to generate the images

    <Note>
      Make sure your `.env` file contains a valid `GEMINI_API_KEY` before this step. If you have not configured it yet, see [post-installation setup](/installation#post-installation-setup).
    </Note>
  </Step>

  <Step title="Receive your deliverables (Phase 5)">
    Claude delivers a final set of assets:

    - **Interactive HTML showcase page** — all SVG variants with hover effects, suitable for sharing with stakeholders
    - **SVG files** — editable vector source files for each variant
    - **PNG exports** — transparent-background rasters at 1024×1024px (or your requested size)
    - **Showcase images** — four AI-generated presentation images with your logo on professional backgrounds

    The showcase images are ready to drop into a landing page, pitch deck, or README.
  </Step>
</Steps>
