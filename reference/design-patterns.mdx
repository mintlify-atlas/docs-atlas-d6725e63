Every SVG logo generated by this skill is built from a defined library of pattern types. This reference documents the full catalogue — the philosophy behind each pattern, its SVG implementation, suitable product types, and how patterns combine into more complex compositions. Use it when evaluating generated variants or when providing targeted feedback to refine a design direction.

## SVG technical requirements

All patterns in this library follow the same technical baseline:

- **`viewBox="0 0 100 100"`** — coordinates are expressed as logical units out of 100, which makes positioning intuitive and scaling lossless at any size
- **`currentColor`** — all `fill` and `stroke` values use `currentColor` so the parent container controls the logo color via CSS
- **`<defs>`** — reusable elements (dots, clips) are defined once and referenced with `<use href="#id">`
- **`<clipPath>`** — used for masking dots or lines to geometric boundaries
- **Geometric primitives preferred** — `<circle>`, `<rect>`, `<line>`, `<polygon>` over complex `<path>` where possible
- **Line weight range**: primary strokes use `stroke-width` of 2.5–4; secondary elements use 1–2
- **Negative space**: at least 40–50% of the canvas should be empty

## Dot matrix patterns

<Tabs>
  <Tab title="Concentric circles">

The concentric ring layout arranges dots radially outward from a deliberately empty center. The center void creates tension between presence and absence. Decreasing dot size on outer rings adds depth.

**Key parameters**: 2–4 rings, dots per ring following a ratio (6 / 12 / 18), dot radius decreasing outward (3 → 2.5 → 2), consistent angular spacing.

**Best use cases**: AI platforms, data tools, origin or source concepts, scientific products.

Rounded rectangles convey modularity and structure. Their softened corners make them friendlier than sharp squares while retaining order and precision. Removing selected dots from the grid creates a recognizable shape or initial letter.

**Key parameters**: 3×3, 4×4, or 5×5 grid; corner radius at 30–50% of the shorter side; consistent gaps; selective dot removal for a readable shape.

**Best use cases**: Tech platforms, developer tools, modular systems, dashboard products.

## Geometric shape patterns

<AccordionGroup>
  <Accordion title="Concentric rings">

Multiple rings of different stroke widths create depth and layering. Adding a filled center dot anchors the composition and provides a clear focal point.

**Best use cases**: Iteration, cycles, data processing, completeness concepts.

```svg

    <circle cx="50" cy="50" r="35" stroke-width="2"/>

    <circle cx="50" cy="50" r="15" stroke-width="1"/>

    <circle cx="50" cy="50" r="4" fill="currentColor" stroke="none"/>
  </g>
</svg>
```

  </Accordion>
  <Accordion title="Arc segments">

Broken circles create dynamism and suggest motion or progress. Varying the arc span (270°, 180°) and combining arcs at different radii builds rhythm.

**Best use cases**: Loading states, progress trackers, circular navigation concepts.

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <g fill="none" stroke="currentColor" stroke-width="3" stroke-linecap="round">
    <!-- Outer arc: 270 degrees -->
    <path d="M 50 15 A 35 35 0 1 1 15 50"/>
    <!-- Inner arc: 180 degrees, rotated -->
    <path d="M 70 50 A 20 20 0 1 1 30 50"/>

    <circle cx="50" cy="50" r="4" fill="currentColor" stroke="none"/>
  </g>
</svg>
```

  </Accordion>
  <Accordion title="Boolean intersection">

When two shapes overlap, the clipped intersection creates a new, unexpected form. This represents synthesis, combination, or emergence. A `<clipPath>` isolates the intersection area cleanly.

**Best use cases**: Collaboration platforms, merging or integration services, API gateways.

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <clipPath id="circle-clip">
      <circle cx="58" cy="50" r="22"/>
    </clipPath>
  </defs>
  <g fill="currentColor">
    <!-- Left circle outline -->
    <circle cx="42" cy="50" r="22" fill="none" stroke="currentColor" stroke-width="2"/>
    <!-- Right circle outline -->
    <circle cx="58" cy="50" r="22" fill="none" stroke="currentColor" stroke-width="2"/>
    <!-- Intersection area (filled, low opacity) -->
    <circle cx="42" cy="50" r="22" clip-path="url(#circle-clip)" fill="currentColor" opacity="0.15"/>
  </g>
</svg>
```

  </Accordion>
  <Accordion title="Nested polygons">

Shapes within shapes create visual depth. Rotating inner polygons relative to outer ones introduces dynamism and a layered system feel.

**Best use cases**: Security, stability, layered architectures, frameworks.

## Line system patterns

<AccordionGroup>
  <Accordion title="Horizontal lines with circle mask">

Parallel lines filling a circular boundary represent data flow, code structure, or ordered content within a container. Varying line thickness creates emphasis and hierarchy. The `<clipPath>` masks the lines cleanly at the circle boundary.

## Node network patterns

Node network patterns represent connections, relationships, and systems. Primary nodes are larger; secondary nodes are smaller and lower-opacity. Curved edges (quadratic Bezier paths) add elegance over straight lines.

**Best use cases**: Agent frameworks, network infrastructure, graph databases, social platforms, connectivity products.

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <g>
    <!-- Edges (curved connections) -->
    <path d="M 30 70 Q 50 70, 50 50" stroke="currentColor" stroke-width="2" fill="none"/>
    <path d="M 50 50 Q 50 30, 70 30" stroke="currentColor" stroke-width="2" fill="none"/>
    <path d="M 30 70 Q 20 40, 35 35" stroke="currentColor" stroke-width="1.5"
          fill="none" opacity="0.5"/>
    <path d="M 35 35 Q 50 35, 50 50" stroke="currentColor" stroke-width="1.5"
          fill="none" opacity="0.5"/>
```

## Letter and symbol integration

<AccordionGroup>
  <Accordion title="Geometric letter abstraction">

Extract the structural DNA of a letter using pure geometric elements — not a rendered font. The letter "P" below uses a vertical stem line and a semicircular bowl, with an accent dot at the stem base.

**Key principle**: Capture the essential geometry, not the full letterform. The result should be abstract enough to function as a mark while remaining recognizable.

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <g fill="none" stroke="currentColor" stroke-width="3" stroke-linecap="round">
    <!-- Vertical stem -->
    <line x1="35" y1="25" x2="35" y2="75"/>
    <!-- Bowl (semicircle) -->
    <path d="M 35 25 L 50 25 A 15 15 0 0 1 50 55 L 35 55"/>
    <!-- Accent: dot at stem base -->
    <circle cx="35" cy="75" r="3" fill="currentColor" stroke="none"/>
  </g>
</svg>
```

  </Accordion>
  <Accordion title="Dot matrix letter">

## Animation patterns (web display only)

SVG animations add life to logos in web contexts — interactive HTML showcases, landing pages, and README files. The two patterns below are the ones used in the skill's generated variants. For static exports (PNG, print), Claude strips animations automatically.

**Continuous rotation:**

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <g>
    <animateTransform attributeName="transform" type="rotate"
      from="0 50 50" to="360 50 50" dur="20s" repeatCount="indefinite"/>
    <!-- logo content rotates here -->
    <circle cx="50" cy="35" r="3" fill="currentColor"/>
    <circle cx="65" cy="50" r="3" fill="currentColor"/>
    <circle cx="50" cy="65" r="3" fill="currentColor"/>
    <circle cx="35" cy="50" r="3" fill="currentColor"/>
  </g>
</svg>
```

**Breathing effect (opacity pulse):**

```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="8" fill="currentColor">
    <animate attributeName="opacity" values="0.3;1;0.3"
      dur="3s" repeatCount="indefinite"/>
  </circle>
</svg>
```

To request an animation during Phase 3 iteration, specify the duration and element explicitly — for example: `Add a slow rotation to the outer dot ring, one full rotation every 20 seconds, rotating around center point (50, 50).`

## Reusable definitions reference

When you need to place the same element many times, define it once in `<defs>` and reference it with `<use>`:

```svg
<defs>
  <!-- Define once, use many times -->
  <circle id="dot" r="3" fill="currentColor"/>
  <clipPath id="circle-bound">
    <circle cx="50" cy="50" r="35"/>
  </clipPath>
</defs>

<!-- Reuse the dot element -->
<use href="#dot" x="30" y="40"/>
<use href="#dot" x="50" y="40"/>
<use href="#dot" x="70" y="40"/>
```

<Tip>Keep the `viewBox` at `0 0 100 100` for all logos. This coordinate system makes positioning intuitive — values map directly to percentages of the canvas — and ensures lossless scaling to any output size.</Tip>
